## 3. 调用时机与回退

- 首次加载：
  - 读取当前配置：`GET /sys/ts-user-voice-config/current`
  - 读取推荐音色：`GET /sys/ts-voice-profiles`
- 条件筛选变化（性别/年龄）：
  - 重新请求推荐音色列表
- 点击“试听音色”：
  - 先保存配置，再请求试听音频
- 失败回退：
  - 推荐音色列表失败 -> 使用页面 fallback；
  - 试听返回无音频地址 -> 提示已保存但无可播放音频；
  - 我的音色库能力未对接 -> 提示缺口，不调用危险接口。

## 4. 后端接口缺口咨询（待你确认后实施）

当前缺口：页面存在“我的音色库 + 重命名/删除”交互，但后端现有接口语义偏公共音色管理，直接复用有误删风险。

### 4.1 缺口接口 A：我的音色列表
- 用途：支撑 `sound-edit` 的“我的音色库”tab 数据源。
- 请求：`GET /sys/ts-user-voice-profiles`
- 鉴权：需要登录态（与当前 `/sys` 业务接口一致）。
- 入参（query）：
  - `pageNo: number`（可选，默认 1）
  - `pageSize: number`（可选，默认 20）
  - `keyword: string`（可选）
  - `status: number`（可选，默认 1）
- 出参（核心）：
  - `records: Array<{ id: number; name: string; gender?: string; ageGroup?: string; tags?: Array<{id:number;tagName:string}>; createdAt?: string }>`
  - `total/current/size/pages`
  - 错误码：沿用 JEECG `success=false` + `message`
- 前端映射：
  - `myVoices <- records`
- 调用时机：
  - 进入页面后或切换到“我的音色库”tab 时加载；
  - 支持手动重试。
- 兼容与回退：
  - 未上线前继续 fallback 并提示“我的音色库暂未对接”。

### 4.2 缺口接口 B：我的音色重命名
- 用途：支持“我的音色库”卡片菜单“重命名”。
- 请求：`PUT /sys/ts-user-voice-profiles/{id}`
- 鉴权：需要登录态，且必须校验归属用户。
- 入参（body）：
  - `name: string`（必填，1-50）
- 出参（核心）：
  - `{ id, name, updatedAt }`
  - 错误码：音色不存在、无权限、名称非法
- 前端映射：
  - 成功后更新 `myVoices` 对应项 `name`
- 调用时机：
  - 用户确认重命名后提交，成功后局部刷新。
- 兼容与回退：
  - 未上线前仅提示，不改数据。

### 4.3 缺口接口 C：删除我的音色
- 用途：支持“我的音色库”卡片菜单“删除”。
- 请求：`DELETE /sys/ts-user-voice-profiles/{id}`
- 鉴权：需要登录态，且必须校验归属用户。
- 入参：path `id`
- 出参（核心）：
  - 成功消息或 `{ success: true }`
  - 错误码：音色不存在、无权限、被引用不可删
- 前端映射：
  - 成功后从 `myVoices` 移除对应项
- 调用时机：
  - 二次确认后调用，失败可重试。
- 兼容与回退：
  - 未上线前仅提示，不触发删除动作。
