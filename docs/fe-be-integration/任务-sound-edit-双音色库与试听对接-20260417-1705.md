# Task: sound-edit Voice Library + Preview Integration

- Page: `src/app/pages/sound-edit/index.tsx`
- Created at: `2026-04-17 17:05`

- Goal: connect recommend/my voice libraries to backend list APIs, keep default as unselected voice, and ensure preview generates playable audio without relying on R2 upload.

## 1. Confirmed APIs

- Recommend voice library list: `GET /sys/ts-voice-profiles`
- My voice library list: `GET /sys/ts-user-voice-profiles`
- Voice preview generate: `POST /sys/ts-voice-profiles/preview`
- Voice config load/save: `GET/PUT /sys/ts-user-voice-config/current`

## 2. Field Mapping

- Recommend card list <- `ts-voice-profiles.records[]`
- My voice card list <- `ts-user-voice-profiles.records[]`
- Card name <- `name`
- Card tags <- `tags[].tagName` (fallback to gender/ageGroup)
- Preview request voice identity <- `providerVoiceId` + optional `voiceProfileId`
- Preview playable url <- `previewAudioUrl`

## 4. Risks and Rollback

- Risks:
  - Some voice profiles may not have `providerVoiceId`, causing preview failure.
  - Large preview payload (data URL) can increase response size when upload is off.
- Rollback:
  - Frontend: rollback only `sound-edit` page + `ts-voice` API wrapper.
  - Backend: rollback only `TsVoiceProfileServiceImpl.previewVoice` fallback logic.
