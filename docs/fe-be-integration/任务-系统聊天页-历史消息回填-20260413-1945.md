- 目标：进入系统会话后，根据 `sessionId` 拉取并显示历史消息，不改动页面布局结构。

## 页面字段到接口字段映射
- `message.id <- TsChatMessage.id`
- `message.role <- TsChatMessage.senderType`（`user` 映射用户气泡，其它映射 AI 气泡）
- `message.content <- TsChatMessage.contentText`

## 调用时机与失败回退
- 首次进入页面且 `sessionId` 有效时触发加载：
  - `GET /sys/ts-chat-messages?sessionId={id}&pageNo=1&pageSize=100`
- 后端列表为倒序，前端 `reverse()` 后按时间正序显示。
- 接口失败时保留页面默认消息，避免布局抖动。
