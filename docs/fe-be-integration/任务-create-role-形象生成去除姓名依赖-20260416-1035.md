# 任务：create-role 形象生成去除姓名依赖（2026-04-16）

## 1. 背景与目标
- 页面：`src/app/pages/create-role/components/create-character.tsx`
- 目标：修复“形象生成依赖名字”的问题，未填写名字时也可发起形象生成。
- 边界：仅改对接逻辑与调用策略，不改 UI 布局/样式结构。

## 2. 任务拆分
| 任务 | 状态 | 说明 | 证据 |
| --- | --- | --- | --- |
| T1 生图调用策略调整 | 已完成 | 无名字时不再强制创建角色草稿，直接调用 `POST /sys/ts-roles/one-click-image` 同步生图 | `src/app/pages/create-role/components/create-character.tsx` |
| T2 保留原异步链路 | 已完成 | 有 `roleId` 或有名字时继续沿用原异步生图+轮询详情逻辑 | `src/app/pages/create-role/components/create-character.tsx` |

## 3. 字段映射与调用时机
- 调用接口：`POST /sys/ts-roles/one-click-image`
- 请求字段：
  - `roleId`：有草稿角色时传；无则不传
  - `roleName`：有名字时传；无则不传
  - `gender/occupation/backgroundStory`：按页面现有输入透传
  - `asyncGenerate`：
    - 有 `roleId` 或有名字：`true`
    - 无 `roleId` 且无名字：`false`

## 4. 回退与风险
- 回退：若生图失败，保持页面现有字段与布局不变，仅提示错误信息。
- 风险：无名字场景走同步链路，返回 `generateRecordId` 概率较低；前端已兼容“有记录轮询/无记录直接取图”两种返回。

## 5. 本轮结论
- 已实现“形象生成不依赖名字”。
- 未改动任何 UI 布局结构，仅调整请求分支逻辑。
