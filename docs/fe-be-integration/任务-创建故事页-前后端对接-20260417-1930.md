# 任务：创建故事页前后端对接（2026-04-17）

- 页面：`src/app/pages/create-story/index.tsx`
- API 封装：`src/lib/api/ts-story.ts`
- 状态：进行中（本轮已完成主链路）
- 约束：不修改现有 UI 布局与视觉结构，仅改数据对接与状态逻辑

## 一、目标

1. 接通 `create-story` 的故事保存链路（新增/编辑）。
2. 接通故事设定、场景设定、剧情大纲三个一键生成接口。
3. 章节模式下接通章节保存（新增/编辑）。
4. 保持页面布局不变，按钮行为改为真实接口调用。

## 二、接口映射

### 1) 页面字段 -> 故事保存接口 `POST/PUT /sys/ts-stories`

- `activeTab` -> `storyMode`（`normal` / `chapter`）
- 故事设定输入框 -> `storySetting`
- 场所设定输入框 -> `sceneNameSnapshot`
- 故事背景（内部状态）-> `storyBackground`
- 普通模式剧情大纲 -> `remark`
- 角色绑定：本轮按产品要求暂不传，后续单独接入
- AI 生成标记 -> `isAiStorySetting` / `isAiOutline`

说明：
- 页面无独立标题输入框，保存时按以下顺序生成标题：
  1. 已有 `storyTitle`
  2. `storySetting / outlineText` 截断
  3. `未命名故事-时间戳`

### 2) 一键生成

- 故事设定生成：`POST /sys/ts-stories/story-setting-generate`
  - 入参来源：`storyId/title/storyMode/storyIntro/storySetting/storyBackground/ideaInput`
  - 回填：`title/storyIntro/storyMode/storySetting/storyBackground`

- 场景生成：`POST /sys/ts-stories/story--scene-generate`
  - 入参来源：`title/storyMode/storySetting/storyBackground/sceneSetting`
  - 回填：`sceneSummary`（兜底 `sceneNameSnapshot`）到场所输入框

- 大纲生成：`POST /sys/ts-stories/story--outline-generate`
  - 入参来源：`storyId/title/storyMode/storySetting/sceneSetting/storyBackground/chapterCount`
  - 回填：
    - 普通模式：将章节结果拼接为大纲文本
    - 章节模式：覆盖章节卡片列表

### 3) 章节保存

- 新增：`POST /sys/ts-story-chapters`
- 编辑：`PUT /sys/ts-story-chapters`
- 字段映射：
  - `chapterNo/chapterTitle/chapterDesc/openingContent/missionTarget/openingRoleId/forbiddenRoleIds/sortNo`
