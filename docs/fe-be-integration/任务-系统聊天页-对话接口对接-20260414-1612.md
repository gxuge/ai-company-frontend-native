# 任务-系统聊天页-对话接口对接-20260414-1612

## 任务背景
- 页面：`pages/admin-chat`
- 目标：仅对接“聊天历史 + 聊天发送/AI回复”接口。
- 边界：不处理语音、文件、相机、图片、通话、菜单等其它模块。
- UI约束：不修改现有布局与视觉结构，只改数据请求与状态逻辑。

## 字段映射（计划）
- 历史消息：
  - `TsChatMessage.senderType` -> `ChatMessage.role`
  - `TsChatMessage.contentText` -> `ChatMessage.content`
  - `TsChatMessage.id` -> `ChatMessage.id`
- 发送回复：
  - 入参：`sessionId`、`userContent`、`historyCount`
  - 回参：`contentText` -> AI消息内容；`assistantMessageId` -> AI消息ID（若无则前端兜底）

## 风险与回退
- 风险1：`sessionId` 缺失导致无法发送。
  - 回退：前端给出提示并不发请求。
- 风险2：接口失败导致对话中断。
  - 回退：插入失败提示消息，不改动布局。

## 验收标准
- 能加载后端历史消息并渲染到现有气泡区。
- 发送文本后调用 `/sys/ts-chat-sessions/ai-reply`，返回内容出现在 AI 气泡。
- 推荐问题点击后走同一发送链路。
- 全程不修改页面布局样式结构。
